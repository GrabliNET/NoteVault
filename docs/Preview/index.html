Возможности

Всё что нужно для заметок

От быстрой записи мысли до экспорта полноценного документа.

Редактор Markdown

Split View: редактор слева, HTML-превью справа. Автосохранение каждые 2 секунды. Поддержка таблиц GFM, task lists, блоков кода.

Теги и фильтрация

Назначайте теги через удобный диалог или прямо в front-matter. Заметка с одним тегом хранится в папке с его именем — вы найдёте нужное даже без приложения.

Книги

Объединяйте заметки в книги с фиксированным порядком. При чтении все главы склеиваются в один документ. Экспорт всей книги в PDF — одним нажатием.

Переносимое хранилище

Vault — обычная папка. Скопируйте её на флешку, загрузите в Dropbox или откройте в VS Code. Формат файлов совместим с Obsidian.

Экспорт в PDF

Экспортируйте одну заметку или целую книгу в PDF с форматированием заголовков, таблиц, кода и цитат. Встроенные изображения из вложений включаются автоматически.

Вложения

Прикрепляйте файлы и изображения через кнопку 📎. Ссылки вставляются синтаксисом ![[файл.png]] — изображения отображаются в превью сразу.

Как работает

Четыре шага до результата

1

Выберите папку

Укажите любую папку на диске как vault. NoteVault создаст нужную структуру автоматически.

2

Создайте заметку

Нажмите Ctrl+N, введите заголовок и теги. Файл .md появится в нужной папке.

3

Организуйте

Добавляйте теги, объединяйте в книги, прикрепляйте файлы. Всё синхронизируется в real-time.

4

Экспортируйте

Одна заметка или вся книга — в PDF одним нажатием. Или просто откройте .md файл в другом редакторе.

Хранение данных

Двойной подход к данным

Заметки хранятся как .md файлы для переносимости, а SQLite обеспечивает быстрый поиск и связи.

Markdown файл

Источник истины для содержимого

Формат .md с YAML front-matter
Содержит Текст, теги, книга, порядок
Читается Любым текстовым редактором
Совместимость Obsidian, VS Code, Typora

SQLite база данных

Индексы, связи и быстрый поиск

Файл _db/notes.db
Таблицы notes, tags, books, attachments
Назначение Поиск, фильтрация, счётчики
При потере MD-файлы сохраняются целыми

О проекте

Почему NoteVault?

Большинство приложений для заметок хранят данные в проприетарных форматах или требуют постоянного подключения к интернету. NoteVault построен на противоположном принципе: ваши данные — ваши, в открытом формате, на вашем диске.

Разработан как учебный проект в рамках курса по системам контроля версий. Весь код задокументирован через Doxygen с генерацией DocBook XML документации.
