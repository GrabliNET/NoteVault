Подробное описание

Менеджер физической структуры директорий внутри vault.

Инкапсулирует все правила размещения файлов: какой заметке соответствует какой путь, где лежат вложения, как избежать коллизий имён.

Структура vault

vault/
├── _db/
│   └── notes.db          ← SQLite база данных
├── _unsorted/
│   └── *.md              ← заметки без тегов
├── _multi/
│   └── *.md              ← заметки с двумя и более тегами
├── _attachments/
│   └── <note-slug>/
│       └── <file>        ← вложения конкретной заметки
└── <tag>/
    └── *.md              ← заметки ровно с одним тегом

Правила размещения

Число тегов

Директория

0

_unsorted/

1

<tagname>/

2+

_multi/

Эта логика позволяет пользователю ориентироваться в хранилище даже без приложения: тематические заметки лежат в папках с понятными именами.

Заметки

Класс работает только с путями; чтение/запись файлов — ответственность com.notevault.service.VaultService.

См. также. com.notevault.service.VaultService

noteFilePath()

Path com.notevault.util.VaultLayout.noteFilePath (Note note)

Вычисляет ожидаемый абсолютный путь к MD-файлу заметки.

Путь строится по схеме: <vaultRoot>/<folder>/<slug>.md, где <folder> определяется тегами, а <slug> — санированным заголовком.

Предупреждения

Возвращаемый путь может уже существовать. Для гарантии уникальности используйте uniquePath(Path).

sanitizeFileName()

String com.notevault.util.VaultLayout.sanitizeFileName (String title)[static]

Преобразует заголовок заметки в безопасное имя файла (без расширения).

Алгоритм:

  1. Обрезает пробелы.

  2. Заменяет символы, недопустимые в именах файлов, на _.

  3. Заменяет пробелы на -.

  4. Приводит к нижнему регистру.

  5. Сжимает множественные дефисы.
