Подробное описание

Модель заметки — центральная сущность приложения.

Каждая заметка соответствует одному файлу в формате Markdown внутри хранилища (vault). Путь к файлу хранится в поле filePath относительно корня vault, что обеспечивает переносимость хранилища.

Структура файла заметки

Файл начинается с YAML front-matter блока, за которым следует тело заметки:

---
tags:
- java
book:
- java_guide
order: 3
---
# Заголовок заметки
Текст заметки...

Размещение файла в vault

Физическое расположение файла определяется количеством тегов:

  • 0 тегов → _unsorted/<slug>.md
  • 1 тег → <tag>/<slug>.md
  • 2+ тегов → _multi/<slug>.md

◆ getBodyContent()

String com.notevault.model.Note.getBodyContent ( )

Возвращает тело заметки без YAML front-matter блока.

Если файл начинается с ---, ищет закрывающий --- и возвращает всё после него. Если front-matter отсутствует — возвращает content целиком.

Возвращает
Текст заметки без front-matter, или пустая строка если content равен null.
