Подробное описание

Центральный сервисный слой приложения.

Координирует все операции, требующие одновременного изменения файловой системы и базы данных. Является единственным местом, где принимаются решения о физическом расположении файлов.

Принцип двойного хранения

Каждая заметка существует в двух представлениях:

  • MD-файл — источник истины для содержимого и базовых метаданных (теги, книга, порядок — через YAML front-matter).
  • SQLite — источник истины для быстрого поиска, индексов, связей и агрегированных данных (счётчик тегов и т.д.).

При любом изменении оба представления обновляются в рамках одного публичного метода сервиса.

Жизненный цикл vault

VaultService.getInstance().openVault(path); // 1. выбор vault
Note n = service.createNote("Title", "", tags); // 2. создание заметки
service.saveNote(n, newContent); // 3. сохранение
service.deleteNote(n); // 4. удаление

Note com.notevault.service.VaultService.createNote ( String title,
String body,
List< Tag > tags ) throws Exception

Создаёт новую заметку с заданным содержимым и тегами.

Алгоритм:

  1. Определяет целевую директорию по тегам через VaultLayout#noteFilePath.
  2. Генерирует уникальное имя файла (добавляет суффикс при коллизии).
  3. Строит полный текст файла (front-matter + тело) через FrontMatterParser#buildContent.
  4. Записывает файл на диск.
  5. Сохраняет метаданные в БД через NoteDAO#insert.
Аргументы
titleЗаголовок заметки.
bodyТело заметки в формате Markdown (без front-matter).
tagsСписок тегов (может быть пустым).
Возвращает
Созданная и сохранённая заметка.
Исключения
Exceptionпри ошибке файловой системы или БД.

◆ getBookContent()

String com.notevault.service.VaultService.getBookContent ( Book book) throws Exception

Формирует объединённый Markdown-текст всех заметок книги.

Заметки берутся в порядке order_index и конкатенируются через горизонтальную линию ---. Физические файлы не изменяются. Результат используется для отображения книги в режиме чтения и для экспорта в PDF.

Формат результата:

# Название книги
Описание книги...
<содержимое заметки 1>
---
<содержимое заметки 2>
Аргументы
bookКнига с корректным id.
Возвращает
Markdown-строка с объединённым содержимым.
Исключения
Exceptionпри ошибке чтения файлов или запроса к БД.
