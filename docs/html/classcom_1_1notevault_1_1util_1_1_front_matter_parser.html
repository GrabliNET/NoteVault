Подробное описание

Парсер и генератор YAML front-matter блока в MD-файлах.

Обеспечивает двунаправленную конвертацию между YAML front-matter внутри Markdown-файлов и полями объекта Note.

Поддерживаемые поля front-matter

Поле Тип Описание
tags список строк Теги заметки
book список строк Имя книги (берётся первый элемент)
order целое число Позиция в книге (с нуля)

Пример front-matter

---
tags:
- java
- tutorial
book:
- java_guide
order: 2
---

Синтаксис вложений

Ссылки на прикреплённые файлы используют Obsidian-style синтаксис:

![[image.png]]
![[document.pdf]]

extractAttachmentRefs(String) выделяет имена файлов из таких ссылок.

Разбирает front-matter файла и заполняет поля объекта Note.

Если front-matter отсутствует или не содержит нужных полей, поля объекта note не изменяются.

Аргументы
contentПолное содержимое MD-файла (front-matter + тело).
noteОбъект заметки, в который записываются распознанные поля.

◆ buildContent()

String com.notevault.util.FrontMatterParser.buildContent ( Note note,
String body )
static

Строит полный текст MD-файла: front-matter (из полей Note) + тело.

Front-matter генерируется вручную (без SnakeYAML) для контроля форматирования вывода и минимизации лишних полей. Если у заметки нет тегов и она не принадлежит книге — front-matter не добавляется.

Аргументы
noteЗаметка с актуальными тегами, книгой и порядком.
bodyТело заметки (без front-matter).
Возвращает
Полный текст файла, готовый для записи на диск.
